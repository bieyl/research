# 🦞 学术龙虾技术架构文档

---

## 1. 系统概述

学术龙虾采用**本地优先**架构，所有核心数据处理均在用户设备本地完成，确保数据安全与隐私保护。

---

---

### 3.2 关键词提取模块 (keyword_extractor.py)

**功能：** 提取核心关键词，自动学科分类

**技术栈：**
- 关键词提取：TextRank / YAKE
- 学科分类：预训练分类模型

**处理流程：**

```
输入文本 → 分词 → 候选词提取 → 关键词排序 → 学科分类 → 输出
```

---

### 3.3 实验日志模块 (lab_log.py)

**功能：** 录入实验数据，生成规范报告

**技术栈：**
- 语音识别：SpeechRecognition / Vosk (离线)
- 数据解析：正则表达式 + NLP
- 报告生成：Jinja2 模板

**处理流程：**

```
语音/文本输入 → 数据解析 → 结构化存储 → 模板填充 → 报告生成
```

**数据库设计：**

```sql
CREATE TABLE experiment_logs (
    id INTEGER PRIMARY KEY,
    date TEXT NOT NULL,
    purpose TEXT,
    raw_data TEXT,
    structured_data TEXT,  -- JSON 格式
    analysis TEXT,
    suggestions TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

---

### 3.4 PPT 大纲模块 (ppt_outline.py)

**功能：** 根据研究进展生成 PPT 结构

**技术栈：**
- 内容理解：NLP 模型
- 大纲生成：规则模板 + AI 生成

**处理流程：**

```
输入研究进展 → 内容分析 → 匹配模板 → 生成大纲 → 输出
```

**输出格式：**

```json
{
  "title": "研究进展汇报",
  "slides": [
    {
      "page": 1,
      "title": "标题页",
      "points": ["研究进展汇报", "姓名、日期"],
      "chart_type": null
    },
    {
      "page": 2,
      "title": "研究背景",
      "points": ["问题定义", "研究意义"],
      "chart_type": "text"
    }
  ]
}
```

---

### 3.5 参考文献格式化模块 (reference_formatter.py)

**功能：** 转换参考文献格式

**技术栈：**
- 文献解析：正则表达式 + NLP
- 格式转换：规则引擎

**支持格式：**
- GB/T 7714 (中国国家标准)
- APA (美国心理学会)
- IEEE (电气电子工程师学会)
- MLA (现代语言协会)
- Chicago (芝加哥格式)

---

### 3.6 语音交互模块 (voice_interface.py)

**功能：** 语音指令识别与执行

**技术栈：**
- 语音识别：SpeechRecognition / Vosk
- 指令解析：规则匹配
- 语音合成：pyttsx3 (离线) / Azure TTS (在线)

**支持指令：**

| 指令 | 触发功能 |
|------|----------|
| "总结这篇论文" | 文献摘要 |
| "生成实验日报" | 实验日志 |
| "导出参考文献" | 参考文献格式化 |
| "打开文献列表" | 文献管理 |

## 4. 数据存储设计

### 4.1 数据库结构

```
academic_lobster.db
├── papers (文献库)
│   ├── id
│   ├── title
│   ├── authors
│   ├── abstract
│   ├── keywords
│   ├── category
│   ├── file_path
│   └── created_at
│
├── experiment_logs (实验日志)
│   ├── id
│   ├── date
│   ├── purpose
│   ├── raw_data
│   ├── structured_data
│   └── created_at
│
├── ppt_templates (PPT 模板)
│   ├── id
│   ├── name
│   ├── structure
│   └── updated_at
│
└── settings (用户设置)
    ├── key
    ├── value
    └── updated_at
```

### 4.2 加密方案

- 数据库加密：SQLCipher (AES-256)
- 文件加密：cryptography 库
- 密钥管理：操作系统密钥环 (keyring)

## 5. 安全架构

### 5.1 数据隔离

```
用户 A 设备                    用户 B 设备
┌─────────────┐              ┌─────────────┐
│ 学术龙虾     │              │ 学术龙虾     │
│ ┌─────────┐ │              │ ┌─────────┐ │
│ │数据库 A  │ │              │ │数据库 B  │ │
│ │(加密)   │ │              │ │(加密)   │ │
│ └─────────┘ │              │ └─────────┘ │
└─────────────┘              └─────────────┘
      ↓                            ↓
  无数据共享                    无数据共享
```

### 5.2 权限控制

| 权限 | 用途 | 是否必需 |
|------|------|----------|
| 文件读取 | 读取 PDF/文档 | 是 |
| 麦克风 | 语音输入 | 否 (可选) |
| 网络访问 | 云端 AI 推理 | 否 (可离线) |
| 通讯录 | - | 不申请 |
| 位置 | - | 不申请 |
| 相机 | - | 不申请 |
