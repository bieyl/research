---

## 1. 安全承诺

学术龙虾将数据安全与隐私保护作为产品设计的核心原则，从架构源头杜绝数据泄露风险。

---

## 2. 数据安全架构

### 2.1 本地优先原则

```
┌─────────────────────────────────────────────┐
│              用户设备（本地）                │
│  ┌─────────────────────────────────────┐    │
│  │           学术龙虾客户端             │    │
│  │  ┌─────────┐  ┌─────────┐          │    │
│  │  │文献数据库│  │实验日志库│  (SQLite) │    │
│  │  │(AES-256)│  │(AES-256)│  加密存储  │    │
│  │  └─────────┘  └─────────┘          │    │
│  │         ↓ 本地处理，不上传           │    │
│  │  ┌─────────────────────────────┐    │    │
│  │  │      AI 推理引擎             │    │    │
│  │  │  本地模式：完全离线          │    │    │
│  │  │  云端模式：仅脱敏文本        │    │    │
│  │  └─────────────────────────────┘    │    │
│  └─────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
```

**核心原则：**
- ✅ 所有原始数据存储于用户本地设备
- ✅ 不上传至任何云端服务器
- ✅ 用户完全掌控自己的数据

## 3. 权限最小化设计

### 3.1 权限申请清单

| 权限类型 | 申请状态 | 用途说明 | 是否必需 |
|----------|----------|----------|----------|
| 文件读取 | ✅ 申请 | 读取用户主动选择的 PDF/文档 | 是 |
| 文件写入 | ✅ 申请 | 保存生成的报告、摘要 | 是 |
| 麦克风 | ⚠️ 可选 | 语音输入功能 | 否 (可关闭) |
| 网络访问 | ⚠️ 可选 | 云端 AI 推理 (可选) | 否 (可离线) |
| 通讯录 | ❌ 不申请 | 与科研无关 | - |
| 位置信息 | ❌ 不申请 | 与科研无关 | - |
| 相机 | ❌ 不申请 | 与科研无关 | - |
| 通话记录 | ❌ 不申请 | 与科研无关 | - |
| 短信 | ❌ 不申请 | 与科研无关 | - |

### 3.2 权限使用说明

**文件读取权限：**
- 仅读取用户主动选择的文件
- 不扫描用户整个文件系统
- 不访问与科研无关的目录

**麦克风权限：**
- 仅在语音模式启用时激活
- 语音识别后可立即关闭
- 支持离线语音识别 (Vosk)

**网络访问权限：**
- 可配置为完全离线模式
- 云端模式下仅发送脱敏文本
- 不传输原始文件内容

## 4. 数据加密方案

### 4.1 数据库加密

- **加密算法：** AES-256
- **加密库：** SQLCipher
- **密钥管理：** 操作系统密钥环 (keyring)

```python
# 数据库加密示例
import sqlite3
import keyring

class EncryptedDatabase:
    def __init__(self, db_path: str):
        self.db_path = db_path
        self.password = self._get_password()
    
    def _get_password(self) -> str:
        # 从系统密钥环获取加密密码
        password = keyring.get_password("academic_lobster", "db_password")
        if password is None:
            password = self._generate_password()
            keyring.set_password("academic_lobster", "db_password", password)
        return password
    
    def connect(self) -> sqlite3.Connection:
        # 使用 SQLCipher 加密连接
        conn = sqlite3.connect(self.db_path)
        conn.execute(f"PRAGMA key = '{self.password}'")
        return conn
```

### 4.2 文件加密

- **加密算法：** AES-256-GCM
- **加密库：** cryptography
- **加密范围：** 敏感实验数据、未发表研究内容

## 5. 行为透明机制

### 5.1 操作日志

所有 AI 操作均记录于本地日志，用户可随时查阅：

```
[2026-03-12 10:30:15] 用户上传论文：resnet.pdf
[2026-03-12 10:30:45] 生成摘要完成，耗时 30 秒
[2026-03-12 10:31:00] 摘要已保存到数据库
[2026-03-12 10:35:20] 用户输入实验数据 (语音)
[2026-03-12 10:35:25] 生成实验日报完成
```

**日志位置：** `~/.academic_lobster/logs/operation.log`

### 5.2 网络监控

显示所有网络请求的目标地址与内容摘要：

```
网络请求记录：
[2026-03-12 10:30:40] POST https://api.example.com/summarize
  发送内容：脱敏文本片段 (200 字符)
  接收内容：摘要结果
  状态：成功
```

**查看方式：** 设置 → 网络监控 → 查看请求历史

### 5.3 数据导出

一键导出所有数据：

- 文献数据库 (SQLite)
- 实验日志 (Markdown/CSV)
- 操作日志 (TXT)
- 用户设置 (JSON)

**导出方式：** 设置 → 数据管理 → 导出全部数据

### 5.4 数据删除

一键清空所有数据：

- 删除数据库
- 删除缓存文件
- 删除操作日志
- 删除用户设置

**删除方式：** 设置 → 数据管理 → 清空全部数据

**注意：** 删除后数据不可恢复，请谨慎操作。

## 6. 合规承诺声明

### 6.1 数据安全承诺

> 学术龙虾承诺：所有用户数据（文献、实验记录、研究内容）均存储于用户本地设备，不上传至任何云端服务器。AI 推理可选本地模式或云端模式，云端模式下仅发送脱敏后的文本片段，不传输原始文件。用户可随时导出、删除全部数据。

### 6.2 隐私保护承诺

> 学术龙虾不会收集、存储、传输任何用户身份信息、研究内容或实验数据。我们尊重每一位研究者的隐私权，产品设计遵循隐私优先 (Privacy by Design) 原则。

### 6.3 知识产权承诺

> 学术龙虾尊重文献版权，仅提供摘要与索引服务，不提供全文下载。用户上传的论文仅限个人学习研究使用，不得用于商业目的。

## 8. 安全最佳实践

### 8.1 用户建议

1. **启用设备加密**
   - Windows: 启用 BitLocker
   - macOS: 启用 FileVault
   - Linux: 启用 LUKS

2. **定期备份数据**
   - 每周导出实验日志
   - 备份到外部硬盘或加密云存储

3. **使用强密码**
   - 数据库密码至少 12 位
   - 包含大小写字母、数字、符号

4. **及时更新软件**
   - 开启自动更新
   - 关注安全公告

### 8.2 课题组建议
