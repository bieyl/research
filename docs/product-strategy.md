# 🦞 学术龙虾 产品战略重构

## 一、市场调研：现有科研工具痛点

### 1. 文献管理工具（Zotero/Mendeley）
- ❌ 只管理文献，不与实验数据关联
- ❌ 云端同步，未发表研究有泄露风险
- ❌ 中文支持差，标签体系不符合中国科研习惯

### 2. AI 论文工具（ChatPDF/SciSpace）
- ❌ 必须上传 PDF 到云端
- ❌ 单次对话，无法形成知识积累
- ❌ 无法与实验记录关联

### 3. 实验记录工具（LabArchives/Notebook）
- ❌ 订阅制，年费$100+
- ❌ 云端存储，数据不可控
- ❌ 功能臃肿，学习成本高

### 4. 写作工具（Grammarly/Quillbot）
- ❌ 仅支持英文
- ❌ 无法理解科研语境
- ❌ 不涉及中文论文写作

## 二、学术龙虾差异化定位

### 核心差异化 1：科研知识图谱 ⭐ 独家功能

**痛点：** 研究生读了几百篇论文，但知识是碎片化的，无法形成体系。

**解决方案：**
- 自动构建"问题 - 方法 - 结论"知识图谱
- 论文与实验数据双向关联
- 一键生成"领域知识地图"

**示例：**
```
研究问题：图像识别准确率提升
├── 方法分支 1：更深的网络
│   ├── ResNet (2015) → 残差连接
│   ├── DenseNet (2017) → 密集连接
│   └── 你的实验 → 改进的残差块
├── 方法分支 2：注意力机制
│   ├── SENet (2018) → 通道注意力
│   └── CBAM (2018) → 空间 + 通道
└── 方法分支 3：数据增强
    ├── Mixup (2017)
    └── CutMix (2019)
```

### 核心差异化 2：实验 - 文献智能关联 ⭐ 独家功能

**痛点：** 做实验时忘记哪篇论文支持这个假设，写论文时找不到支撑实验结果的文献。

**解决方案：**
- 录入实验数据时，自动推荐相关论文
- 阅读论文时，自动关联历史实验
- 写论文时，一键插入"相关实验数据"

**示例：**
```
【实验记录】2026-03-10
实验：ResNet 改进，准确率 87% → 89%

📚 关联文献（自动推荐）：
1. ResNet 原始论文 - 残差块设计原理
2. Identity Mappings in Deep ResNets - 改进的残差块
3. 你的历史实验 - 2026-02-15 类似尝试

💡 智能提示：
- 你的改进与 Identity Mappings 论文中的"pre-activation"相似
- 建议在论文中引用该论文并对比
```

### 核心差异化 3：组会汇报一键生成 ⭐ 独家功能

**痛点：** 每周组会要花 2-3 小时准备 PPT，但内容都是已有的实验记录和文献。

**解决方案：**
- 自动聚合本周实验记录 + 阅读的论文
- 生成完整 PPT（含图表、数据对比）
- 支持导出为 PPTX 文件

**示例：**
```
输入：选择时间范围（2026-03-05 ~ 2026-03-12）

输出：
- P1 标题页
- P2 本周工作概览（自动统计：3 个实验，5 篇论文）
- P3-5 实验进展（自动插入实验数据表格和图表）
- P6-7 文献阅读（自动插入论文摘要）
- P8 问题与讨论（从实验记录的"问题"字段提取）
- P9 下周计划
```

### 核心差异化 4：本地课题组协作 ⭐ 独家功能

**痛点：** 云端协作工具数据不可控，本地共享又太麻烦。

**解决方案：**
- 基于局域网的 P2P 数据同步
- 数据仍存储在各白电脑，只同步变更
- 支持"公开/组内/私密"三级权限

**示例：**
```
张师兄的电脑 (192.168.1.100)
├── 公开：课程笔记、教程
├── 组内：实验记录、组会 PPT
└── 私密：未发表数据、论文草稿

你的电脑 (192.168.1.101)
├── 同步张师兄的"公开"和"组内"内容
├── 自己的数据三级分类
└── 离线可用，联网自动同步
```

### 核心差异化 5：中文科研场景优化 ⭐ 独家功能

**痛点：** 现有工具都是英文优先，中文论文写作支持差。

**解决方案：**
- 支持 GB/T 7714 参考文献格式（中国标准）
- 中文论文结构模板（摘要→引言→方法→实验→结论）
- 中文学术用语润色（而非简单翻译）

## 四、技术架构优化

### 性能问题解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 生成慢 | 每次重新计算 | 增加缓存层（SQLite + 内存缓存） |
| 响应慢 | 同步处理 | 异步任务队列（Celery） |
| 界面卡顿 | 前端渲染 | 分页加载 + 虚拟滚动 |

### 新架构设计

```
┌─────────────────────────────────────────────────────────┐
│                    用户界面层                              │
│  Web 界面 (Flask) / 桌面应用 (PyQt) / 命令行              │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                    API 服务层                              │
│  REST API + WebSocket (实时更新)                         │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                    业务逻辑层                              │
│  • 文献服务  • 实验服务  • 知识图谱服务  • 推荐服务       │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                    数据访问层                              │
│  • SQLite (本地数据)  • Redis (缓存)                      │
└─────────────────────────────────────────────────────────┘
```

## 五、参赛展示策略

### 3 分钟演示脚本（重构版）

**开场（15 秒）：**
> "在座的评委老师可能都带过研究生。您有没有遇到过这种情况：学生读了两年论文，问他这个领域有哪些方法，他说不清楚。为什么？因为知识是碎片化的。"

**痛点演示（30 秒）：**
> 展示一个研究生桌面：50 个 PDF 文件、10 个 Excel 实验数据、20 个 PPT 文件，互相独立，无法关联。

**解决方案（90 秒）：**
> 1. 导入 10 篇论文 → 自动生成知识图谱（视觉冲击）
> 2. 录入实验数据 → 自动关联相关论文（智能推荐）
> 3. 点击"生成组会 PPT" → 完整 PPT 自动生成（一键生成）

**价值总结（45 秒）：**
> "学术龙虾不是又一个文献管理工具，而是科研知识的大脑。它帮你把碎片化的知识组织起来，把重复的劳动自动化，让你专注于真正的创新。"

## 六、核心指标

| 指标 | 现有工具 | 学术龙虾目标 |
|------|----------|--------------|
| 文献→知识转化 | 手动整理 2 小时/篇 | 自动构建 30 秒/篇 |
| 实验 - 文献关联 | 无 | 智能推荐 |
| 组会 PPT 准备 | 2-3 小时/次 | 5 分钟/次 |
| 数据安全性 | 云端存储 | 本地优先 |
| 中文支持 | 弱 | 深度优化 |
